### `lib/useEscapeBack.ts` (new)
A naive document `keydown` listener would fire **while Escape is closing a dropdown** and navigate away — the page is full of Radix Selects/Comboboxes/MultiSelects. So the hook skips when:

The overlay check reads the **DOM, not the event**, which holds regardless of listener order: React unmounts on a later tick, so the layer is still mounted while the handler runs. Wired into `BugDetailPage`.

## Backend — multi-value filters
- **`shared/utils/query-array.util.ts`** (new) — `TransformQueryArray()` normalizes `?x=a`, `?x=a,b` and `?x=a&x=b` to `string[]`; blank → `undefined`. **Backwards compatible**: a single value becomes a 1-item array, so `$in` ≡ the old equality match for existing callers.
- `query-bug.dto` / `query-task.dto` — status, severity, assigneeId, projectId, roadmapId, roadmapItemId → arrays with `@IsEnum(..., { each: true })`; validation still rejects junk (`?severity=bogus` → **400** ✓).
- Both repositories → `{ $in: [...] }`. Bug search also gained the **regex escaping + `_id` match** the task repo already had.

### Two real bugs found by verifying against the API rather than trusting the code
1. **axios would have silently broken every multi-select.** Axios 1.x serializes arrays as `?status[]=a` by default, and this API's query parser doesn't read the bracket form — `?severity[]=low&severity[]=medium` returned **all 4 bugs** (filter dropped). Fixed globally in `lib/api.ts` with `paramsSerializer: { indexes: null }` → repeated keys (`?status=a&status=b`), which I'd verified works. No existing caller sends arrays, so nothing else changes.
2. **"Unassigned" filtered nothing.** Unassigned is stored as `''`, but `?assigneeId=` is indistinguishable from "filter not set" (and the transform drops blanks) — it returned **all 3** bugs. Added a documented sentinel (`UNASSIGNED_QUERY = '__unassigned__'` + `resolveAssignees()`) that the repos map back to `''`; the frontend exports the matching `UNASSIGNED`.

### Bug 1 — every card menu click navigated away *(pre-existing, not mine)*
`ProjectCard`'s `<article>` navigates on click. Radix portals the menu out to `<body>`, so DOM-wise it isn't nested — but **React bubbles synthetic events up the component tree, not the DOM tree**. So picking *any* item fired the article's `onClick`. Proof from the instrumented timeline:

```

569ms  - input REMOVED (DIV)
card markup now: no article      <- whole page gone: it navigated
```

**Fix:** `onClick={(e) => e.stopPropagation()}` on `DropdownMenu.Content` in **`Menu.tsx`** — at the root, since `BugsBoardPage` nests a menu in a clickable card too.

### Bug 2 — the menu stole focus back and closed the editor *(mine)*
`@radix-ui/react-focus-scope/dist/index.mjs:87-99` — on unmount it runs `setTimeout(…, 0)` then `focus(previouslyFocusedElement)` **unless the event is defaultPrevented**. | | with fix | fix disabled |
|---|---|---|
| t=50ms | focused | focused |
| t=200ms | focused | **gone, activeElement=BUTTON** ← focus stolen to the trigger |

## Repro + đo, không đoán
Dựng board bug, drag bằng puppeteer, đo rect overlay vs con trỏ:
```
grab "test" (300,213) · card rect x:260 y:199
cursor (600,433) · overlay (300,220) 0x0   ← wrapper 0×0!
OFFSET ERROR: dx=-260 dy=-198
```
**Sai số = đúng bằng toạ độ gốc của card (260,199)**, và overlay nằm đúng tại **delta** tính từ (0,0). Nghĩa là dnd-kit tưởng initial rect = `0×0 @ (0,0)` → đặt overlay ở `0,0 + delta` thay vì `initialRect + delta`.

**Nguyên nhân**: `isDragging && 'hidden'` → `display:none` → `getBoundingClientRect()` trả **toàn số 0**. dnd-kit đo node bị kéo **sau** lần render bật `isDragging`, nên đo trúng lúc node đã `display:none`.

**Không phải do restyle của mình**: `git show HEAD` → dòng `isDragging && 'hidden'` **giống hệt** trong commit cuối. Bug có sẵn, đã commit. Loại trừ luôn giả thuyết "transformed ancestor phá `position:fixed`" — probe đi ngược cây cha: **không có** transform/filter/will-change nào, wrapper đúng là `position: fixed`.

## Fix — sửa **timing**, giữ nguyên hành vi

Chốt: `useEffect` + `requestAnimationFrame` → frame đầu chỉ `opacity-0` (**còn trong layout để dnd-kit đo được**), frame sau mới `hidden` (sập slot). Được cả hai.
