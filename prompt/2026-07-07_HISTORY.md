## Prompt
> this product use mongodb like /Volumes/DEVELOP/GITHUB/ME/product-os/old-report

(Translation: switch product-hub's backend persistence to MongoDB, matching `old-report`'s database engine.)

### Decisions (via AskUserQuestion)
- **Keep multi-tenant** architecture (not reverting to old-report's single-tenant model) — tenancy model is a separate axis from the database engine.
- **MongoDB runs locally via Docker** (not Atlas/cloud) — same style as the Postgres setup it replaces.

### What was done
Migrated the backend from **TypeORM + PostgreSQL** to **Mongoose 8 + MongoDB 7**, touching **infrastructure only** — the ports/adapters DDD boundary meant `application/**` (domain entities, use-cases, dtos, mappers) and `presentation/**` (controllers, guards, filters) needed zero changes.

- **ID strategy:** kept `UniqueEntityID` (uuid v4 string) as the domain identity, stored as Mongo's `_id` typed `String` (uuid default) instead of ObjectId — so ids stay opaque strings end-to-end, API contracts and the frontend are unaffected.
- **Removed:** `typeorm`, `@nestjs/typeorm`, `pg` deps + `typeorm`/`migration:*` scripts; `libs/core/infrastructure/database/typeorm/**`; `src/infrastructure/database/typeorm/**`.
- **Added:** `mongoose`, `@nestjs/mongoose`; `libs/core/infrastructure/database/mongoose/base/repository.base.ts` (new `BaseRepository<Entity, Doc>`); `src/infrastructure/database/mongoose/mongoose.module.ts` (`MongooseInfrastructureModule`, reads `MONGODB_URI`).
- **Rewrote** (same ports, new adapters — now the template for every future slice): tenants + users schemas/repositories/modules (Mongoose `Schema`, camelCase fields, `$regex` search replacing SQL `ILIKE`); `app.module.ts` wiring; `docker-compose.yml` (`mongo:7` replacing `postgres:16-alpine`); `.env`/`.env.example` (single `MONGODB_URI`); `dev.sh` readiness check (`mongosh --eval "db.adminCommand('ping')"` replacing `pg_isready`).
- Conventions borrowed from reading `old-report/models/{User,Project}.ts`: Mongoose 8 schemas, `{ timestamps: true }`, unique/indexed fields, case-insensitive regex search.

### Verification (all passed)
1. `docker compose up -d db` → Mongo ping OK.
2. `npm run test` — existing `create-user.use-case.spec.ts` passed **unchanged**, proving the application layer never touched Mongo specifics.
3. `npm run start:dev` booted clean; smoke sequence: register (tenant+admin) → login → `GET /auth/me` → duplicate email → 409 → bad password → 401 → invalid payload → 400 — all correct.
4. Inspected Mongo directly: `_id` fields are uuid strings (not ObjectId) on both `users`/`tenants` collections, confirming the ID strategy.
5. Multi-tenant isolation: registered a 2nd tenant, confirmed `GET /users` only returns that tenant's own user.
6. `./dev.sh` full cycle: Mongo readiness wait, both servers boot, clean shutdown, ports 3000/3001 freed.

Complexity: simple-to-medium, mechanical, zero regressions — the DDD/hexagonal architecture from the original request paid off exactly as intended.
