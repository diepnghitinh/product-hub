## Prompt
> fix `{ "statusCode": 500, "message": "The specified bucket does not exist", "error": "NoSuchBucket", "path": "/v1/uploads" }`

Once storage **is** configured (S3/S3-compatible), uploading to a bucket that doesn't exist made the AWS SDK throw `NoSuchBucket`. That's not a Nest exception, so it surfaced as a raw **500**.

## Work done
- **`backend/src/infrastructure/storage/storage.service.ts`** — `uploadS3` now **provisions the bucket on first use**: on a `NoSuchBucket` `PutObject`, it runs `CreateBucketCommand` then retries the put once. AWS needs a `LocationConstraint` for every region but `us-east-1` (S3-compatible endpoints ignore it), so it's set only when there's no custom endpoint and region ≠ us-east-1. `BucketAlreadyOwnedByYou`/`BucketAlreadyExists` races are treated as success; if creation genuinely fails, it throws an **actionable `BadRequestException`** ("bucket … couldn't be created — create it in your provider, or fix Settings → Storage") instead of a 500.
