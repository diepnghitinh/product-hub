## Prompt

Two auth capabilities. Checked in on the two UX forks before building UI — user chose: reset = *type-or-generate + copy*; self-change = *user-menu → dialog*.

## Work done
**Backend — new admin-reset endpoint** (mirrors `ChangePasswordUseCase` minus the current-password check):
- `application/users/dtos/reset-password.dto.ts` — `newPassword @IsString() @MinLength(6)`.
- `application/users/use-cases/reset-user-password.use-case.ts` — tenant-scoped find (`!user || user.tenantId !== tenantId` → "not found"), `password.hash` → `user.changePassword` → `users.update`. Registered in barrel + `ApplicationUsersModule`.
- `presentation/users/users.controller.ts` — `@Patch(':id/password')` `@Roles(Role.ADMIN)`, `tenantId` from `@AuthUser()` (never the body). Distinct path from `@Patch(':id')`, so no route collision.

**Frontend**
- `features/users/api.ts` — `useResetUserPassword` (`PATCH /users/:id/password`) + `useChangeMyPassword` (`PUT /users/me/password`).
- `features/admin/ResetPasswordDialog.tsx` (new) — type or **Generate** (crypto, look-alike-free, one-of-each-class + shuffle) + **Copy**; on success a confirm state keeps the password on screen to relay. Wired into `AdminPeoplePage` via a **kebab `Menu`** (Reset password · Remove) replacing the lone trash button.
- `features/account/ChangePasswordDialog.tsx` (new) — current/new/confirm, client-side mismatch + min-length guards, `toast` on success. Opened from the sidebar footer **user menu → Change password**.
